# JavaScript Data Transformation Practice

This repository contains structured JavaScript problems focused on mastering **manual data transformation**, **loop-based logic**, and **backend-style processing**.

The goal is to build strong fundamentals required for **Node.js APIs** and **React state handling** without relying on higher-order methods.

---

## 📌 Focus Areas

- Loop-based logic (`for` loops only)
- Object-based data mapping (hashmaps)
- Nested data traversal
- Manual grouping and aggregation
- Backend-style data shaping
- Tree and hierarchical data construction
- Multi-step transformations
- Real-world data processing patterns

---

## 🚫 Constraints

To strengthen core logic skills, the following are **not used**:

- `.map()`, `.filter()`, `.reduce()`
- Arrow functions
- Destructuring
- Built-in sort methods

All solutions are written using:

- `for` loops
- Arrays (`push`)
- Objects (dynamic keys)
- Basic conditionals

---

## 🧠 Problem Categories

### 1. Data Transformation
- API → UI shaping
- Derived fields
- Conditional mapping

### 2. Grouping & Aggregation
- Group by user, date, or category
- Build summary objects

### 3. Nested Data Processing
- Arrays inside objects
- Objects inside arrays

### 4. Frequency & Counting
- Track occurrences
- Multi-level counting

### 5. Tree / Hierarchical Data
- Parent → children mapping
- Comment trees

### 6. Combined Logic Problems
- Group + filter + transform
- Sorting + extraction
- Parsing + aggregation

---

## 🎯 Learning Goals

- Build **predictable, bug-free logic**
- Improve **loop control and indexing**
- Develop **backend-ready thinking**
- Handle **real-world data structures**
- Reduce dependency on shortcuts

---

## ⚠️ Common Mistakes Targeted

- Incorrect indexing (`i`, `j`)
- Resetting state inside loops
- Mixing initialization and update logic
- Misusing objects vs arrays
- Wrong placement of push/return
- Confusing counting vs tracking max

---

## 🚀 How to Use

1. Pick a problem
2. Solve using only allowed tools
3. Focus on:
   - Step-by-step logic
   - Correct state tracking
4. Verify using test cases
5. Debug using console logs if needed

---

## 📈 Difficulty Progression

Problems are designed to:

- Increase gradually (~10%)
- Avoid repetition of patterns
- Introduce **one new challenge at a time**
- Focus on **depth, not new concepts**
